## üìñ Overview
This project contains the logic and configuration for an **Apigee Edge API Proxy** designed to act as a middleware transformation layer.

It intercepts raw JSON alert payloads from upstream monitoring systems, parses critical threshold violations, and reconstructs the payload into a **Microsoft Teams MessageCard** format. This ensures that operational alerts sent to Teams are readable, actionable, and formatted correctly (using HTML and Markdown) before they reach the webhook.

## üèó Architecture & Request Flow
The proxy logic utilizes Apigee's **PreFlow** hooks to manipulate the payload before it is sent to the backend target.

`Monitoring Tool` ‚ûî `Proxy Endpoint` ‚ûî **[ JS Transformation ]** ‚ûî **[ Header Assignment ]** ‚ûî `Target Endpoint` ‚ûî `MS Teams`

## ‚öôÔ∏è Implementation Guide
Because Apigee relies on XML configuration to trigger JavaScript resources, this section details the specific Policy definitions and Flow steps required to implement this solution.

### 1. Policy Definitions
You must define two policies: one to execute the logic, and one to assign the resulting payload.

**A. JavaScript Policy (`msteams-alert-integration.xml`)**
*Role: Executes the JS file located in the resource repository.*
```xml
<?xml version="1.0" encoding="UTF-8" standalone="yes"?>
<Javascript async="false" continueOnError="false" enabled="true" timeLimit="200" name="msteams-alert-integration">
    <DisplayName>msteams-alert-integration</DisplayName>
    <Properties/>
    <ResourceURL>jsc://msteams-alert-integration.js</ResourceURL>
</Javascript>
```

**B. Assign Message Policy (`msteams-alert-header.xml`)**
*Role: Takes the variable `{teamsPayload}` (generated by the JS script) and sets it as the request body.*
```xml
<?xml version="1.0" encoding="UTF-8" standalone="yes"?>
<AssignMessage async="false" continueOnError="false" enabled="true" name="msteams-alert-header">
    <DisplayName>msteams-alert-header</DisplayName>
    <AssignTo createNew="false" type="request"/>
    <Set>
        <Payload>{teamsPayload}</Payload>
    </Set>
</AssignMessage>
```

### 2. Flow Execution (Proxy Endpoint)
The script is triggered in the **PreFlow** of the Proxy Endpoint. This ensures the request is intercepted immediately upon entry.

**Proxy Endpoint XML Configuration:**
```xml
<ProxyEndpoint name="default">
    <PreFlow name="PreFlow">
        <Request>
            <Step>
                <Name>msteams-alert-integration</Name>
            </Step>
        </Request>
        <Response/>
    </PreFlow>
    <Flows/>
    <PostFlow name="PostFlow">
        <Request/>
        <Response/>
    </PostFlow>
    <HTTPProxyConnection>
        <BasePath>/msteams-space-proxy-prod</BasePath>
        <VirtualHost>secure</VirtualHost>
    </HTTPProxyConnection>
    <RouteRule name="default">
        <TargetEndpoint>default</TargetEndpoint>
    </RouteRule>
</ProxyEndpoint>
```

### 3. Flow Execution (Target Endpoint)
Before the request leaves Apigee to hit the Microsoft Teams Webhook, we apply the final payload assignment in the Target Endpoint **PreFlow**.

**Target Endpoint XML Configuration:**
```xml
<TargetEndpoint name="default">
    <PreFlow name="PreFlow">
        <Request>
            <Step>
                <Name>msteams-alert-integration</Name>
            </Step>
            <Step>
                <Name>msteams-alert-header</Name>
            </Step>
        </Request>
        <Response/>
    </PreFlow>
    <Flows/>
    <PostFlow name="PostFlow">
        <Request/>
        <Response/>
    </PostFlow>
    <HTTPTargetConnection>
        <URL>[https://outlook.office.com/webhook/YOUR_WEBHOOK_URL](https://outlook.office.com/webhook/YOUR_WEBHOOK_URL)</URL>
    </HTTPTargetConnection>
</TargetEndpoint>
```

## üíª Script Logic Details
The `msteams-alert-integration.js` script performs the following operations:
1.  **Context Validation:** Checks for `PROXY_REQ_FLOW` to ensure logic only runs on the request path.
2.  **Extraction:** Parses `request.content` to retrieve specific metrics (Duration, Status Code, Threshold Value).
3.  **Formatting:** Uses HTML tags (`<br>`) to format the text for the Teams card renderer, as standard `\n` is often ignored by Teams cards.
4.  **Variable Setting:** Stores the final JSON string into `context.setVariable("teamsPayload", ...)`, which is consumed by the XML policies.

## üöÄ Deployment Checklist
To deploy this logic to an Apigee Edge organization:
1.  **Upload Resource:** Upload `msteams-alert-integration.js` to the `apiproxy/resources/jsc` folder.
2.  **Create Policies:** Create the two XML policies listed above in the `apiproxy/policies` folder.
3.  **Configure Endpoints:** Add the `<Step>` configurations to the **PreFlow** in both Proxy and Target endpoints default XMLs.
4.  **Deploy:** Save as a new revision and deploy to the `prod` environment.
